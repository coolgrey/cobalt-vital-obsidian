# CocFlow AI — 솔루션

## 핵심 개념
환자가 자연어로 입력한 증상을 AI가 분석하여,
임상 기준에 맞는 구조화된 정보로 변환하고,
의료진이 즉시 활용 가능한 형태로 제공한다.

> "환자의 증상을 상담하는 것이 아니라, 의료진이 사용할 수 있는 데이터로 변환하는 것"

## 서비스 플로우

```
환자 자연어 입력
→ 증상 해석 및 구조화 (임베딩 기반)
→ 근거 기반 정보 결합 (RAG)
→ 의료진 확인 (구조화 요약 자동 생성)
→ 환자 전달 (맞춤형 증상 대응 가이드)
```

## HIRA 공공데이터 활용

| HIRA 데이터 | 활용 방식 |
|------------|----------|
| 상병코드(KCD) + 진료내역 | 증상-질환 연관 정보 구성, 임상적 해석 정확도 향상 |
| 의약품 처방·급여 데이터 | 항암제 부작용 관련 증상 정보 보완 |
| 요양기관 정보 | 지역 의료 연계 구조 지원 |

## 차별화 포인트

| 기존 서비스 | CocFlow AI |
|-----------|-----------|
| 설문 기반 입력 | 자연어 기반 증상 처리 |
| 정보 제공으로 끝남 | 의료진 workflow 중심 설계 |
| 환자-앱 단방향 | 환자 데이터 → 의료진 전달 |
| 단순 상담 챗봇 | 임상 활용 데이터 생성 시스템 |

## 확장 가능성
환자 증상 데이터 누적 → digital phenotype 생성
→ 단순 기록을 넘어 데이터 기반 환자 관리 체계로 발전

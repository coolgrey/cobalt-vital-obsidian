# CocFlow AI — 기술 스택

## AI 기술 구성

| AI 기술 | 역할 | HIRA 데이터 연계 |
|---------|------|----------------|
| 임베딩 기반 자연어 의미 분석 | 환자 다양한 표현 → 정규화된 증상 개념 변환 | KCD 코드 기반 표준화 |
| 증상 유사도 기반 매칭 | 입력 표현 ↔ 임상 개념 연결 | 심평원 상병 분류 체계 |
| RAG 기반 문서 검색 | 해당 증상의 근거 기반 정보 확보 | 심평원 급여 기준·가이드라인 |
| LLM 구조화 요약 생성 | 의료진 활용 가능 형태로 정보 정리 | 처방 정보 기반 복약·생활지침 |

## 임상 기준 적용
- CTCAE (Common Terminology Criteria for Adverse Events) 기반 데이터 구조
- PRO (Patient-Reported Outcomes) 방법론

## 현재 개발 수준
- 주요 증상(설사·속쓰림) 중심 핵심 기능 프로토타입 설계 완료
- 환자 입력 → 증상 분석 → 요약 생성의 기본 흐름 구현 확인
- 핵심 기능 기반 프로토타입 수준

## 단기/중기/장기 계획

| 단계 | 내용 |
|------|------|
| 단기 | 주요 증상 데이터셋 확장·정제, 임상 시나리오 기반 검증 |
| 중기 | 다증상 처리 고도화, 의료진 UI/UX 개선 |
| 장기 | 병원 시스템 연계, FHIR 기반 PHR 연계 데이터 통합 플랫폼 |

## 면책 조항
본 시스템은 의료적 판단을 수행하지 않으며,
의료진의 의사결정을 지원하기 위한 정보 정리 도구이다.

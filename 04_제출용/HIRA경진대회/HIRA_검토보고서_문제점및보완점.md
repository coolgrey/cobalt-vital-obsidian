### 1. 핵심 기술 검증의 과장 / 모호성

**문제점**
```
사업계획서 명시:
"알고리즘 근거: 서울대병원·네이버 공동연구 (JMIR, 2025), 151,281명 검증"
"최근 출시된 서울대병원 + 네이버 공동 연구 (JMIR 저널, 2025)"

현실 확인:
- 실제로 이 논문이 존재하는가?
- 이 논문이 우리의 "3-레이어 가중치 구조"를 검증한 것인가?
- 아니면 일반적인 건강지수 연구이며 우리가 차용한 것인가?
```

**위험성**
- 심평원 심사위원이 논문 검증 시 "과장" 판단 가능 (감점 또는 탈락)
- 근거 없는 주장으로 신뢰도 하락

**보완 방안**
- [ ] JMIR 논문 실제 존재 여부 확인
- [ ] 존재한다면: 정확한 제목, 저자, DOI 명시
- [ ] 우리 알고리즘과의 관계: "참고·인용" 관계 명확히 하기
- [ ] 만약 논문이 우리 알고리즘을 검증한 것이 아니라면:
  - → "서울대병원과의 임상 검증 진행 중" 으로 수정
  - → 현재 진행 상태 명시 (예: "2026년 6월 완료 예정")

**현재 상태**: ⚠️ **긴급 검증 필요**

### 2. ARPU (월 9,900원) 근거 부족

**문제점**
```
사업계획서 명시:
"연간 ARPU 가정: 월 9,900원 × 12개월 = 118,800원
(B2C 구독 + B2B 기업복지 패키지 혼합 기준 추정)"

→ "추정값"이라고 명시했지만, 추정 근거가 없음
```

**위험성**
- 시장 규모 계산 (SAM: 7,700억원)이 이 수치에 완전히 의존
- 오류가 나면 전체 시장 분석 신뢰도 하락
- 심평원: "실제 시장 조사 데이터가 없는가?" 질문 가능

**근거 부재**
- 국내 건강관리 앱 구독료 벤치마크 (당근마켓, 버팀목 등) 언급 없음
- 기업복지 건강관리 서비스 시장 가격 조사 부족
- 글로벌 레퍼런스(싱가포르 Steps, b.well) 수익 구조 분석 부족

**보완 방안**
- [ ] 국내 경쟁사 가격대 조사
  - 카카오헬스케어: 월 가격?
  - 나만의닥터: 월 가격?
  - 기타 건강관리 앱: 월 가격?
- [ ] B2B(기업복지) 벤치마크
  - 일반적인 기업 임직원 건강관리 앱 가격대
  - 직원 1인당 월 단가 시장 평균
- [ ] 글로벌 사례의 수익화 방식 조사
  - b.well: 보험사로부터의 거래 단가?
  - 싱가포르 Steps: 정부 예산?
- [ ] 최종 근거 표시: [벤치마크: ~], [시장조사: ~]

**현재 상태**: ⚠️ **데이터 보충 필요**

### 3. B2G 수주 단가 (건당 2,000~3,500만원) 근거 부재

**문제점**
```
사업계획서 명시:
"B2G 공공기관 | 시범사업 수주 | 건당 2,000~3,500만원 | 1년차 2,000만원"

→ 근거가 전혀 제시되지 않음
```

**위험성**
- 정부 시범사업이 정말 이 가격대로 진행되는가?
- 프로젝트 규모(참여 기관, 지역, 기간)에 따라 가격이 크게 달라질 수 있음
- 심평원: "이 단가는 어디서 나온 수치인가?" 질문 가능

**조사 필요 사항**
- 현재까지 유사한 지자체/보건소 시범사업 사례
- 성남창업센터 네트워크에서 얻을 수 있는 정보

**보완 방안**
- [ ] 실제 정부 시범사업 사례 조사
  - 성남시청 / 판교 구청 건강증진 예산?
  - 지자체 디지털 헬스 시범사업 사례 가격대?
- [ ] 성남창업센터 / 판교창업존 컨택으로부터 정보 수집
- [ ] 보수적 추정값으로 조정
  - "1회 시범사업: 1,500~2,000만원" (보수적)
  - 또는 프로젝트 규모별 가격표 제시

**현재 상태**: ⚠️ **현장 검증 필요**

### 4. 건강정보 고속도로 연동의 현실성 미확인

**문제점**
```
사업계획서 명시:
"0단계: App-to-App FHIR (건강검진 10년치 + 걸음수·수면) | 2026.04~"
"1단계: 건강정보 고속도로 정식 등록 | 2027 상반기"
"1,263개 의료기관 연계"

→ 우리가 실제로 이것을 구현할 수 있는가?
```

**구체적 의문점**
1. **App-to-App FHIR 현실성**
   - 건강검진 데이터를 보유한 기관(서울대병원 등)과 이미 연동 계약이 있는가?
   - 아니면 "예상"하는 것인가?
   - FHIR 파서 개발에 실제로 몇 명월(man-month)이 필요한가?

2. **건강정보 고속도로 정식 등록**
   - 등록 요건이 무엇인가? (법인 등록? 심사? 승인 기간?)
   - 2027년 H1에 정말 가능한가?
   - 등록 후 즉시 1,263개 의료기관 데이터 수신이 가능한가?

3. **의료기관 FHIR 제공 현황**
   - 1,263개 중 실제로 FHIR API 제공 기관은 몇 개인가?
   - 대부분의 소규모 의원은 아직 FHIR 준비가 안 되어있지 않은가?

**보완 방안**
- [ ] 실제 연동 가능 의료기관 확인
  - 건보공단 고속도로 공식 문서에서 현황 파악
  - 실제 API 스펙 확인 (FHIR 표준 수준?)
- [ ] 개발사와의 기술 협의
  - FHIR 파서 개발 난이도 재평가
  - 의료기관별 호환성 이슈 검토
- [ ] 정식 등록 요건 및 일정 명확화
  - 건보공단 공식 요건 문서 확인
  - 실제 등록 기간 조사 (심사 3개월? 6개월?)

**현재 상태**: ⚠️ **기술 실현 가능성 재검증 필요**

### 5. 사용자 온보딩 문제: 동기 부족

**문제점**
```
사업계획서 명시:
"MVP → 자체 포인트 (앱 내 사용)"
"MVP는 수동 입력으로 시작"

→ 사용자가 왜 건강검진 데이터를 수동으로 입력할 동기가 있는가?
```

**현실적 문제**
- 사용자: "귀찮은데, 앱 내 포인트가 뭔가 쓸모 있나?"
- 앱 내 포인트 → 수동 입력 → 포인트 적립 → 앱 내 사용 (완전히 폐쇄적)
- 초기 사용 동기 매우 약함

**개선 필요**
- 포인트 현금화 경로가 얼마나 빨리 확보되는가?
- 초기 인센티브 전략은? (예: 가입 시 보너스 포인트?)
- 소셜 공유 기능 등으로 바이럴 효과를 낼 방법은?

**보완 방안**
- [ ] 베타 사용자 100명 대상 온보딩 경로 구체화
- [ ] 초기 포인트 현금화 파트너 확보 (카드사? 간편결제?)
- [ ] 사용자 리텐션 지표 목표 설정 (예: 30일 사용자 30%, 90일 사용자 10%)
- [ ] 포인트 이외의 동기 분석 (건강 향상도 시각화? 순위표? 커뮤니티?)

**현재 상태**: ⚠️ **UX/PM 검토 필요**

### 6. CocFlow와의 관계 모호

**문제점**
```
모두의창업 사업계획서에는:
"코발트바이탈 = 건강계좌(예방) + CocFlow AI(치료 과정)"

심평원 사업계획서에는:
"건강계좌만 중점" (CocFlow 언급 거의 없음)

→ 심평원 제출은 건강계좌만 하는가? 통합 서비스인가?
```

**의문점**
1. CocFlow가 심평원 경진대회 취지에 맞는가?
   - 심평원: "보건의료빅데이터·AI 활용" → 건강계좌는 ○, CocFlow는?
   - CocFlow는 암 환자 증상 데이터 구조화 → 빅데이터 활용인가?

2. 심평원 제출 vs 다른 공모 제출
   - 건강계좌 → 심평원 (보건의료빅데이터)
   - CocFlow → 의료 AI 경진대회 또는 다른 공모?

**보완 방안**
- [ ] 명확한 포지셔닝 결정:
  - **옵션 1**: 심평원은 건강계좌만, CocFlow는 별도 공모
  - **옵션 2**: 통합으로 제출하면 → 명확한 통합 시나리오 기술
- [ ] 선택 후 사업계획서 일관성 정리

**현재 상태**: ⚠️ **전략 결정 필요**

### 7. 경쟁사 분석 깊이 부족

**문제점**
```
사업계획서 경쟁사 분석:
"나만의닥터(진료) · 카카오헬스케어(질환관리) · 룰루메딕(복약)
→ 모두 수직 연결 서비스. '검진 이후 일상 전구간 장기관리'는 공백."

→ 정말로 이들이 검진 후 예방 구간을 안 하는가?
→ 아니면 사업 모델의 차이인가?
```

**미흡한 점**
1. **나만의닥터**: 비대면 진료도 하지만, 건강검진 후 follow-up도 할 수 있지 않은가?
2. **카카오헬스케어**: 정말 이미 확진자 질환관리만 하는가? 검진 후 예방도 가능하지 않은가?
3. **룰루메딕**: 약물 복용자만 타겟하는가? 건강한 사람도 약국 앱 쓰지 않는가?

**보완 필요**
- 각 경쟁사의 실제 제품/서비스 분석 (앱 설치 후 체험)
- 각사의 공식 포지셔닝 vs 실제 사용자 경험 비교
- 우리가 진짜 "공백"을 채우는가, 아니면 "차별화된 UX"를 제공하는가?

**보완 방안**
- [ ] 경쟁사 5개 이상 상세 분석
  - 제품 기능 맵 (검진, 예방, 질환관리, 진료, 복약)
  - 가격 모델
  - 월 활성사용자 수 (공개된 데이터)
  - 펀딩 규모 및 팀 구성
- [ ] 우리의 차별화 포인트 명확화
  - "검진 이후 예방"이 정말 공백인가? (정량적 증거)
  - 웨어러블 + 건강검진 통합이 차별화인가? (타사는 안 하는가?)
  - 인센티브 구조가 차별화인가? (구체적인 UX 설명)

**현재 상태**: ⚠️ **경쟁사 리서치 심화 필요**

### 8. 규제/법적 리스크 분석 미흡

**문제점**
```
사업계획서 명시:
"디지털의료제품법(2025) 비의료 건강관리 분류 → 즉시 출시 가능"
"건강관리 vs 의료행위 경계 | 높음 | 의료행위 제외 서비스 범위 명확화"

→ 정말로 법적 리스크가 '높음'이면, "즉시 출시 가능"이 아닌 것 아닌가?
```

**구체적 의문점**

| 법적 이슈 | 현재 상태 | 필요한 것 |
|---------|---------|---------|
| 건강 데이터 수집 (FHIR, PHR) | 기술적 준비 O | 법적 승인 절차? |
| 건강검진 수치 기반 "등급 판정" | 당뇨/고혈압 기준 사용 | 이게 의료행위 아닌가? |
| "미션 권고" (운동하세요, 식단 조절) | 일반적 권고 | 특정 환자 맞춤형이면 의료행위 아닌가? |
| 의료진 리포트 제출 | 월 1회 자동 생성 | 의료진 동의 없이 제출 가능한가? |
| 현금 인센티브 | 포인트 → 현금화 | 이게 '금전적 이득 제공'으로 문제 아닌가? |

**보완 방안**
- [ ] 의료법 전문 변호사 자문 (현재: "자문 확보"만 언급)
  - 실제 자문 완료 시점?
  - 자문 내용 요약 별첨?
- [ ] 개인정보보호법 준수 계획 (현재: 거의 언급 없음)
  - PIAS(개인정보영향평가) 계획?
  - 의료정보보호 가이드 준수?
- [ ] 건강검진 데이터 2차 활용 권리
  - 건보공단 PHR에서 수신한 데이터를 우리가 "상업화"할 권리가 있는가?
  - 라이센싱 비용?

**현재 상태**: ⚠️ **법적 자문 완료 후 보충 필요**

### 9. 팀 역량의 객관화 부족

**문제점**
```
팀 소개:
"최지원: UX 경력 16년"
"윤지현: 임상간호사 배경 (미국 RN/OCN 자격)"

→ 구체적 성과/사례가 거의 없음
```

**구체적 의문점**
1. **최지원의 16년 경력**
   - 어느 회사들에서 일했는가?
   - 만들었던 제품들의 성과는?
   - 토스/뱅크샐러드에서 구체적으로 뭘 했는가?
   - 의료기기 앱: 어떤 앱인가? (인슐린 펌프 어플? 모바일 앱?)
   - DTx 임상: 어느 회사? 어느 질환? 결과는?

2. **윤지현의 임상 경력**

### 13. 손익분기점 분석 미흡

**현재**
```
"손익분기점: 월 사용자 5,000명 도달 시 (월 MAU 기준) | 목표: 2027년 내 달성"
```

**부족한 점**
1. 월 5,000명의 근거는?
   - 월 수익: 5,000 × 10,000원(annual) / 12 = 약 4,167만원/월
   - 월 고정비: 얼마인가?
   - 변동비: 얼마인가?

2. 2027년 내 5,000명 도달 현실성은?
   - 2026년 베타에서 200명 확보
   - 2027년에 25배 성장?
   - 이게 가능한가?

**개선 방안**
- [ ] 상세한 손익 시나리오 분석
  ```
  월 수익 = MAU × ARPU / 12
         = 5,000 × 119,000 / 12 = 약 4,958만원/월
  
  월 고정비:
  - 개발사 유지보수: 3,000만원
  - 클라우드 인프라: 500만원
  - 인건비 (최소 2명): 1,000만원
  - 기타 (마케팅, 법무, 회계): 500만원
  = 약 5,000만원/월
  
  손익분기점 = 고정비 / (ARPU - 변동비)
            = 5,000 / (10,000 - 500) ≈ 5,555명/월
  ```

- [ ] 성장 경로의 현실성 검토
  - 200명(2026년 말) → 5,000명(2027년 12월) = 25배 성장
  - 월 성장률: (5000/200)^(1/12) = 약 47% 월간 성장률 필요
  - 이게 가능한가? (비현실적일 수 있음)
  - 현실적 목표: 2027년 1,000명 → 2028년 5,000명?

**현재 상태**: ✔️ **재무 분석 심화 권장**

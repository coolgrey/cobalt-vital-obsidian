## 사업계획서

**사업명:** AI 기반 암환자 증상 구조화 및 의료진 전달 시스템 — CocFlow AI

## 1. 요약

### 창업아이템 소개
CocFlow AI는 항암치료 중 암환자가 자연어로 입력한 증상을 AI가 임상 기준에 맞게 구조화하여, 의료진이 즉시 활용 가능한 데이터로 변환하는 시스템이다.

핵심은 "환자의 증상을 상담하는 것"이 아니라 "의료진이 사용할 수 있는 데이터로 변환하는 것"이다.

RAG 기반 모델을 활용하여 환자 증상에 대한 근거 기반 관리 지침을 제공함으로써, 병원 외 환경에서도 환자 맞춤형 증상 대응이 가능하도록 한다.

### 창업아이템의 독창성
기존 디지털 헬스 서비스는 설문 기반 입력 방식에 의존하여 환자의 자연어 표현을 임상적으로 해석·구조화하지 못한다. CocFlow AI는 자연어 기반 증상 입력 처리 + 임상 기준 기반 정보 구조화 + RAG 기반 근거 문서 연결 + 의료진 전달용 요약 자동 생성을 통합하여 '임상 활용 데이터 생성 시스템'으로 차별화된다.

### 추진 계획
- **단기:** 주요 증상(설사·속쓰림) 중심 시제품 개발 및 임상 시나리오 검증
- **중기:** 다증상 처리 고도화, 의료진 UI/UX 개선
- **장기:** 병원 시스템·FHIR 기반 PHR 연계, digital phenotype 데이터 플랫폼 확장

### 보건의료분야 공공데이터 활용
건강보험심사평가원 보건의료빅데이터개방시스템을 활용한다.

| HIRA 데이터 | 활용 방식 |
|------------|----------|
| 상병코드(KCD) + 진료내역 | 증상-질환 연관 정보 구성, 임상적 해석 정확도 향상 |
| 의약품 처방·급여 데이터 | 항암제 부작용 관련 증상 정보 보완 |
| 요양기관 정보 | 지역 의료 연계 구조 지원 |

### AI기술 활용
- 임베딩 기반 자연어 의미 분석 → 환자 표현을 정규화된 증상 개념으로 변환
- 증상 유사도 기반 매칭 → 입력 표현 ↔ 임상 개념(KCD) 연결
- RAG 기반 문서 검색 → 근거 기반 정보 확보
- LLM 기반 구조화 요약 생성 → 의료진 활용 데이터 생성

### 기대효과
- 의료진의 환자 상태 파악 시간 단축 → 진료 효율성 향상
- 환자-의료진 간 정보 전달 정확성 개선
- 지역 간 의료 정보 격차 완화, 진료 연속성 향상
- 환자 데이터 구조화·축적으로 digital phenotype 확장 가능

## 2. 창업아이템 개요

### 개발 배경
대표자 윤지현은 상급종합병원 조혈모세포이식센터·혈액병원에서 10년 이상 근무하며, 항암 환자의 증상 관리 과정에서 반복되는 구조적 문제를 직접 경험했다.

환자는 자신의 증상을 주관적으로 표현한다. 의료진은 그것을 임상 데이터로 활용하지 못한다. 전화 상담으로 수집된 비정형 정보는 기록에 남지 않는다. 지방 병원을 방문하면 치료 이력 공유가 안 된다.

이 문제를 해결하기 위해 환자 증상을 임상 기준 기반으로 구조화하는 데이터 설계를 수행했으며, RAG 기반 AI 시스템을 직접 설계하고 핵심 기능을 구현하였다.

### 핵심 내용

```
환자 자연어 입력 (예: "밥 먹으면 속이 너무 쓰려요")
         ↓ 임베딩 기반 의미 분석
증상 해석 (속쓰림 → CTCAE Grade 분류)
         ↓ RAG 기반 근거 연결 + HIRA 처방 데이터 보완
구조화된 임상 요약 자동 생성
         ↓ LLM
의료진 전달 리포트 + 환자 맞춤 대응 가이드
```

### 목표 시장
- 항암치료 환자 및 보호자
- 종합병원·상급종합병원·지역 협력 의료기관
- 디지털 헬스케어 및 원격 환자 관리 서비스 시장

## 3. 독창성·발전가능성·완성도

### 차별화 포인트

| 구분 | 기존 서비스 | CocFlow AI |
|------|-----------|-----------|
| 입력 방식 | 설문·선택지 기반 | 자연어 기반 자유 입력 |
| 출력 방향 | 환자에게 정보 제공 | 의료진에게 구조화 데이터 전달 |
| 임상 기준 | 일반적 건강 정보 | CTCAE 기반 임상 분류 |
| AI 활용 | 단순 정보 검색 | RAG + LLM 통합 파이프라인 |

### 발전 가능성
환자 증상 데이터 누적 → **digital phenotype** 생성
→ 시간에 따른 변화 패턴 반영 → 데이터 기반 환자 상태 프로파일
→ ICD·SNOMED 국제 표준 연계 → 해외 시장 확장 가능

### 현재 개발 수준
- 주요 증상 중심 핵심 기능 프로토타입 설계 완료
- 환자 입력 → 증상 분석 → 요약 생성의 기본 흐름 구현 확인
- 핵심 기능 기반 프로토타입 수준 (아이디어 기획 부문 해당)

## 4. 보건의료분야 공공데이터 및 AI기술을 활용한 사업추진 계획

### 단기 (1년차)
- HIRA 상병코드(KCD) 데이터 활용한 증상-질환 연관 정보 구성
- 의약품 처방 데이터 기반 항암제 부작용 증상 정보 데이터셋 구축
- 주요 증상 처리 핵심 기능 시제품 완성 및 임상 시나리오 검증

### 중기 (2년차)
- 다증상 동시 처리 기능 고도화
- 의료진 workflow 최적화 UI/UX 개선
- HIRA 진료내역 데이터 기반 지역별 의료 연계 구조 추가

### 장기 (3년차+)
- 병원 시스템 연계 구조 확장
- FHIR 기반 PHR 연계 데이터 통합 플랫폼
- digital phenotype 데이터 플랫폼으로 발전

### HIRA 공공데이터 구체 활용 방안

| 데이터 | 출처 | 활용 범위 |
|--------|------|----------|
| 환자표본 데이터(HIRA-NPS) | 심평원 빅데이터 | 항암 치료 패턴 학습, 부작용 예측 |
| 상병코드(KCD) | 심평원 | 자연어 증상 → 임상 코드 매핑 표준화 |
| 의약품 처방 데이터 | 심평원 | 항암제별 부작용 증상 정보 보완 |
| 요양기관 정보 | 심평원 | 지역 협력 의료기관 연계 |

## 5. 기대효과

### 산업적·기술적 파급효과
- 비정형 환자 데이터의 임상 활용 가능 데이터로 자동 변환 — 의료 AI 분야 신규 BM
- digital phenotype 플랫폼 확장 → 환자 데이터 기반 정밀의료 기여
- ICD·SNOMED 국제 표준 연계 → 해외 의료 AI 시장 진출 가능

### 사회적 효과 (ESG)
- 환자 안전성 향상 (적시 증상 감지)
- 의료진 업무 효율 개선
- 지역 간 의료 정보 격차 완화
- 수도권 집중 의료 구조 완화 (진료 연속성 강화)

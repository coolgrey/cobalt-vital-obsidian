## □ 사업모델 소개 및 현재 사업화 단계

### 사업모델(BM) 정의

**핵심 개념:** "건강을 자산처럼 관리한다"

건강계좌는 금융 계좌의 구조를 건강에 적용한 예방형 건강관리 플랫폼이다. 공공 의료 마이데이터 레일(건강정보 고속도로) 위에서 건강자산 관리 UX를 구현하는 상위 활용 레이어 서비스로서 PHR·라이프로그·행동 데이터를 통합하여 장기 건강자산 추적과 금융 인센티브로 연결한다.

**사업모델 정립 배경**

기존 서비스(나만의닥터·카카오헬스케어·룰루메딕)는 비대면 진료·질환관리 등 특정 행위 중심이며, 검진 이후 일상 전구간 건강자산 장기관리 영역은 구조적으로 공백이다. 건강데이터를 자산처럼 해석·관리하는 서비스 필요성이 크다.

**사업모델 검증 정도**
- 12슬라이드 피치덱 완성 및 판교창업존 피칭 완료
- 성남시 분당보건소 건강증진팀 협의 시작 (파일럿 거점 협력 논의 중)
- 외주 개발사 전문 제안서 요청 중 (2026년 5월 계약 예정)
- 예비 파일럿 사용성 인터뷰 계획 (2026년 5월)

### 현재 사업화 단계

**☑ 창업준비 및 초기 아이디어 구체화**
- 서비스 기획(화면설계 23개, Figma 목업 진행 중)

**다음 단계:** 2026년 4월 개발 착수와 동시에 MVP 구현 시작

**차별성:** 인프라 구축 리스크 없이 공공 레일 위에서 상위 활용 레이어(건강자산 관리 UX) 최초 선점

### 기술(제품)의 기능 및 주요특징

**MVP 핵심 기능 (1년차)**

1. **건강정보 수집 (App-to-App FHIR)**
   - 사용자 주도형 데이터 수집
   - 국가건강검진 결과 전항목(최근 10년)
   - Fallback: 나의건강기록 PDF 업로드 옵션

2. **AI 건강자산지수 산출 (핵심 엔진)**
   - 3-레이어 가중치: PHR 50% + 라이프로그 35% + 행동 이행 15%
   - Platinum·Gold·Silver·Care 4단계 등급 + 0~100점 스코어
   - 규칙 기반 초기 모델(1년차) → ML 모델 전환(2년차)

3. **사용자 대시보드 (건강자산 가계부 UX)**
   - 건강자산지수 등급 시각화
   - 건강나이 계산 ("내 몸의 실제 나이는 OO세")
   - 이상 징후 조기 감지 ("최근 수면 패턴이 평소와 다릅니다" 알림)

4. **웨어러블 라이프로그 모듈**
   - MVP: 걸음수·수면 우선 연동
   - 2027년: 심박·산소포화도·스트레스 전항목 고도화

## □ 사업성 및 시장성

### 시장 규모 및 성장성 (TAM·SAM·SOM)

**전체 시장(TAM):** 국내 디지털 헬스케어 전체 약 9조원(2024년) / CAGR 14.9%

**접근 가능 시장(SAM):** Bottom-up 방식 재산출 = 약 3,440억원(보수 시나리오)

- **B2B:** 50인 이상 사업체 35,600개 × 건강복지 예산 편성 40% × 디지털 앱 채택 25% × 100인 × 월 3,000원 × 12개월 = 약 1,280억원
- **B2C:** 연간 건강검진 수검자 1,820만명 × 유료 이용 의향 10% × 월 9,900원 × 12개월 = 약 2,160억원

**확보 가능 시장(SOM):** 초기 3년 내 현실적 공략 목표 = 약 6억원(3년차 누적 매출)
- B2G: 2~3건 수주 기준
- B2B: 30개사 기업복지 계약
- B2C: 3,000명 개인 구독자

### 시장 진입 기회: 제도·기술·소비자 인식 3가지 조건 동시 충족

**①  디지털의료제품법 시행(2025.1.24)**
- 비의료 건강관리 서비스로 즉시 출시 가능
- 인허가 리스크 없음
- 보험사·기업 계약의 법적 근거 확보

**② 건강정보 고속도로(마이헬스웨이) 가동(2023~)**
- 2025년 하반기 기준 상급종합병원 47개소 포함 총 1,263개 의료기관 연계
- 나만의닥터·룰루메딕·카카오헬스케어가 동일 레일에서 이미 출시 완료
- → 기술·제도 실현 가능성이 선례로 증명된 상태
- → 건강계좌는 인프라 구축 리스크 없이 동일 레일 위에서 상위 활용 레이어 구현

**③ 헬스 재테크 소비자 인식 확산**
- 예방·관리중심 헬스케어 패러다임 전환
- 기업 복지 트렌드 변화로 임직원 일상 건강관리 수요 확대
- 기업 복지 헬스케어 시장 연평균 6.4% 성장(한국경영자총협회, 2024)

## 예상 꼬리 질문 3가지

### Q1. 기존 탈락 신청 vs. 신규 신청 — 어떤 개선이 있었는가?

**A:** 기존 신청 탈락의 주요 약점은 (1) SOM 펀넬 로직의 모호성, (2) B2G 공공입찰 수치의 추상성, (3) 기술성 차별화 미흡이었다.

이번 신청에서 보강한 점:
- **SOM 재산출:** Bottom-up 방식으로 B2B(1,280억원) + B2C(2,160억원) = 3,440억원으로 정량화 / 초기 목표 6억원은 SAM 대비 0.17%로 현실적 설정
- **B2G 구체화:** 성남시 보건소 협의(파일럿 거점) + 보건복지부·국민건강보험공단 건강증진 공모 명시 + 2,000~3,500만원/건 단가 근거 제시
- **기술성 강조:** "인프라 리스크 제로"(공공 레일 활용) + "4축 동시 구현 서비스 부재"(경쟁 구도 분석) + "글로벌 검증 모델(b.well, 싱가포르)" 추가

### Q2. 프로토타입 50% 진행 상태 — 실제 개발 일정은 현실적인가?

**A:** 진행 상태를 재정의하면:
- **완료(100%):** 서비스 기획(화면 설계 23개), 기술 구조 분석(FHIR 분석·항목 매핑·파서 설계), 파트너십 준비
- **진행 중(50%):** Figma 목업 제작, SDK 조사
- **미시작(0%):** 실제 코드 개발

**개발 일정의 현실성:**
- **1단계(즉시):** App-to-App FHIR(사용자 주도) → 별도 API 계약 불필요, 법적 적합 경로 확보
- **2단계(2027년):** 건강정보 고속도로 API 정식 연동 → 선례 3개사 출시 완료로 기술·제도 검증됨
- **위험요소:** AI 개발자 채용 실패 시 외주 개발사 계약 3개월 연장 옵션 사전 명시

**외주 개발사 역할:** 소스코드 귀속 계약 필수, 헬스케어 전문사 선정 기준 설정

### Q3. 여성벤처 창업 지원과 다른 정부 지원 프로그램 중복 신청은 안 되는가?

**A:** 다중 신청 전략:
- **예비창업패키지(40M KRW):** 현재 신청 예정 (성남창업센터 입주신청과 동시 추진 가능)
- **초기창업패키지:** 1년차 매출화 후 검토
- **KHIDI(보건산업진흥원) 헬스케어 지원사업:** 별도 공모 확인 후 신청 예정
- **여성벤처 특화 지원:** 대표자(여성) 자격으로 추가 신청 가능

**주의:** 동일 비용 항목의 중복 청구 불가 / 정부 지원금 협약서에 "타 정부사업 동시 수주 불가" 조항 확인 후 진행

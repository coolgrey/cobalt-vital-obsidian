## 1. 나만의닥터 (메라키플레이스)

### 기술

**핵심 인프라**
- 건강정보고속도로 특수전문기관 지정 (2025년 1월)
- 병원·약국 방문 이력, 처방 의약품, 예방접종, 건강검진 결과 등을 한 곳에서 조회
- FHIR 기반 App-to-App 데이터 전송
- AI 홈닥터 기능 (의료 마이데이터 연동)

**제한사항**
- 진료 기록 및 투약 정보 조회 중심 (post-visit)
- 비대면 진료 플랫폼이 주 사업 (의료데이터는 부가 기능)
- 생활 습관 데이터 수집 최소화

### 시장

**사용자 규모**
- 50만명 이용 중
- 2024년 중앙일보 올해의 우수브랜드대상 비대면진료 1위

**비즈니스 모델**
- B2C: 건강지원금 제공 (의료비 인센티브)
- 최저가 병원·약국 연결 수수료
- 의료기관 신환 유입 모델

**수익 특성**
- 비대면 진료 거래 수수료 중심
- 약국·병원 소개료 기반
- 건강데이터는 부가 서비스

### 경쟁상 포지셔닝

| 구분 | 나만의닥터 | 건강계좌 |
|------|---------|--------|
| **주 목적** | 진료 기록 조회 + 비대면 진료 | 일상 전구간 건강 추적 |
| **데이터 범위** | 진료·투약·검진 (과거) | PHR + 라이프로그 + 행동 (현재/미래) |
| **사용자 역할** | 수동적 조회자 | 능동적 관리자 |
| **인센티브** | 의료비 할인 | 건강자산 점수 환원 |
| **고객 가치** | 병원 비용 절약 | 예방 + 금융 이득 |

**나만의닥터 약점**
- 검진 이후 365일 공백 미해결 (Q&A에서 건강계좌가 지적)
- 검진 데이터 활용이 아닌 진료 기록 조회
- 건강 상태 변화 추적 기능 부재

## 2. 카카오헬스케어 (파스타 PASTA)

### 기술

**핵심 인프라**
- 연속혈당측정기(CGM) 센서 연동을 통한 실시간 혈당 데이터 수집
- 음식 촬영 시 자동 분석 (비전 AI)
- 식사 기록, 운동 정보 등 생활습관과 실시간 혈당 추이를 시각화
- 의료진용 대시보드 "파스타 커넥트 Pro"

**기술적 특징**
- SDK 방식 센서 연동 (App-to-App API 아님)
- 삼성헬스, 애플 헬스, 인바디 등 다중 장치 통합
- ML 기반 개인화 리포트 (3D 바디스캔 추가 예정)

### 시장

**사용자 규모**
- 2025년 5월 초 기준 25.8만 다운로드
- 2025년 3월 말 기준 상급종합병원 46곳, 종합병원 51곳 포함 총 331개 병원, 약 6,300명 환자 등록

**타겟 질환**
- 당뇨병 전문화 (제1형, 제2형, 임신성 당뇨)
- 향후 고혈압, 고지혈증, 희귀난치질환으로 확장 계획

**비즈니스 모델**
- B2B2C: 병원 의료진 → 환자 진료
- 보험 급여 연계 (인슐린 펌프 센서 구매 비용 지원)
- 포인트 시스템 (챌린지, 일상 기록 → 카카오페이 포인트 교환)

### 경쟁상 포지셔닝

| 구분 | 카카오헬스케어 | 건강계좌 |
|------|-------------|--------|
| **적용 범위** | 당뇨병 전문 | 비의료 전구간 |
| **데이터 정확도** | CGM 기반 의료 기기 수준 | 검진 + 웨어러블 + 행동 추적 |
| **진료 연계** | 의사 처방 기반 | 공공 보건소 기반 |
| **시작점** | 이미 진단된 환자 | 건강한 개인 + 예방 |
| **임상 검증** | 진행 중 | 파일럿 실증 계획 |

**카카오헬스케어 약점**
- 당뇨 특화로 인한 시장 한정 (적용 질환 확대 중이나 시간 소요)
- 기존 환자 중심 (신규 예방 사용자 확보 어려움)
- 건강정보고속도로 API 연동이 아닌 센서 기반 (개인 측정 기기 필요)

## 3. 룰루메딕 (디스탓 d'stat)

### 기술

**핵심 인프라**
- 의료 마이데이터 선도서비스 사업자 (개인정보보호위원회, KISA, KHIS 주관)
- 국내 유일의 해외 연동형 의료 마이데이터 플랫폼
- AWS 클라우드 기반 인프라 (2025년 7월 협력 강화)
- AI 활용을 위한 의료데이터 구조화, 보안 인증 확보

**데이터 특성**
- 의료 마이데이터 + 재정정보 + 웨어러블 데이터
- 해외 체류 시 의료 컨시어지 서비스 (어시스트카드)
- 바이오 빅데이터 관리 및 분석

### 시장

**사용자 규모**
- 구체적 숫자 미공개 (B2B 중심)

**타겟**
- 의료기관 및 제약사 협업
- 정부 헬스케어 사업 (다중 공공 과제 참여)
- 글로벌 의료 데이터 연계 수요

**비즈니스 모델**
- B2B2C: 제약사, 의료기관 → 환자
- 정부 과제 수주형 매출
- 데이터 활용 기반 부가 서비스

### 경쟁상 포지셔닝

| 구분 | 룰루메딕 | 건강계좌 |
|------|---------|--------|
| **수집 데이터** | 의료 마이데이터 + 국외 연계 | PHR + 일상 행동 + 건강자산지수 |
| **고객층** | 제약사, 의료기관, 정부 | 공공보건소, 기업복지, 개인 |
| **기술 역할** | 데이터 통합 플랫폼 | 예방 관리 및 점수화 |
| **규제 위치** | 의료 마이데이터법 준수 | 비의료 예방 (단계별 검토) |

**룰루메딕 약점**
- B2B 중심 (개인 사용자 직접 접점 부족)
- 의료 데이터 기반이므로 건강한 개인의 일상 행동 추적 미흡
- 해외 연동 강점이 국내 시장에서는 부차적 가치

## 4. b.well (Connected Health)

### 기술

**핵심 인프라**
- 2.2백만 의료제공자, 320개 보험사·랩 데이터 연계
- FHIR 기반 API + 신뢰할 수 있는 의료정보 교환 프레임워크
- 신원 확인, 동의 관리, 데이터 보안, 감시 기능
- ChatGPT Health 파트너 (2026년 1월 발표)

**데이터 특성**
- 의료 기록 + 보험 정보 + 웰니스 앱 통합
- 구조화된 데이터 + 비정형 텍스트 (임상노트 등)
- AI SDK for Health (의료 AI용 데이터 최적화)

### 시장

**사용자 규모**
- ChatGPT 주간 활성 사용자 8억명 중 25%가 주간 건강 관련 질문
- 전 세계 2억3천만명이 주간 건강·웰니스 관련 질문

**타겟**
- AI 플랫폼 회사 (OpenAI, Google, Apple 등)
- 소비자 건강 데이터 통합 수요

**비즈니스 모델**
- B2B2C: AI 회사 → 소비자
- 인프라 제공 모델 (데이터 연결 수수료)
- 2026년 주요 AI 헬스 런칭 다수 포함 (OpenAI, Google, Samsung, Perplexity)

### 경쟁상 포지셔닝

| 구분 | b.well | 건강계좌 |
|------|--------|--------|
| **지역** | 미국 중심 | 한국 특화 |
| **데이터 소스** | 기존 의료 기록 | 건강정보고속도로 + 신규 측정 |
| **고객 구조** | AI 회사 → 소비자 | 공공기관 → 개인 → B2B |
| **기술 위치** | 데이터 연결 (Infrastructure) | 데이터 분석 + 점수화 (Application) |
| **수익화** | API 호출료 | 공공 사업 + 구독료 |

**b.well 특징**
- 글로벌 거대 AI 회사와 협력 (최강 파이프라인)
- 미국 시장 특화 (한국 적용 불가)
- 인프라 중심이므로 건강 관리 기능은 최소

## 6. 핵심 인사이트: 왜 건강계좌가 경쟁할 수 있는가

### 나만의닥터와의 비교
- **나만의닥터**: 의료 기록 조회 + 진료료 절감
- **건강계좌**: 검진 이후 365일 일상 관리 + 건강자산 점수화 + 금융 이득

→ **포지셔닝 차별점**: "검진 이후 공백" 해결

### 카카오헬스케어와의 비교
- **카카오헬스케어**: 당뇨병 환자 혈당 관리
- **건강계좌**: 건강한 개인부터 예방 관리까지 전체 인구

→ **포지셔닝 차별점**: "당뇨 환자가 아닌 일반인 중심"

### 룰루메딕과의 비교
- **룰루메딕**: 의료 마이데이터 플랫폼 (B2B)
- **건강계좌**: 공공 보건소 기반 PoC + B2C 확장

→ **포지셔닝 차별점**: "공공 인프라 위의 개인 맞춤 예방"

### b.well과의 비교
- **b.well**: 미국 의료 기록 통합 + AI 회사 인프라
- **건강계좌**: 한국 건강정보고속도로 + 예방 지수 개발

→ **포지셔닝 차별점**: "한국 공공 레일 위의 지역 맞춤 솔루션"

## 7. 발표 시 강조할 포인트

### "대기업이 진입하기 어려운 이유" (피치 자료 S4.5 재확인)

1. **B2G 구조적 이점**
   - 성남시 보건소 수준 계약은 스타트업이 더 빠르고 유연
   - 나만의닥터도 비대면 진료로 성공했으므로 공공 거버넌스 내 스타트업 진입 선례 존재

2. **데이터 주권 포지셔닝**
   - 카카오·삼성은 자사 생태계 종속 (고객이 센서나 보험료 결정)
   - 건강계좌는 건강정보고속도로 API 중심 (공공 표준)

3. **선점 창 — 지금이 타이밍**
   - 나만의닥터: 진료 중심
   - 카카오헬스케어: 당뇨 중심
   - **건강계좌: 일상 전구간 건강자산 관리 (공백)**

## 8. Q&A 대비 (심사위원 예상 질문)

### Q. "나만의닥터와의 차이는?"
**A**: 나만의닥터는 검진 결과와 진료 기록을 조회하는 도구입니다. 건강계좌는 검진 이후 365일 일상에서 측정되는 건강 데이터(운동, 수면, 음식)를 PHR과 함께 추적하고, 그것을 하나의 점수(건강자산지수)로 환원하는 예방 관리 플랫폼입니다. 비즈니스 모델도 다릅니다 — 나만의닥터는 비대면 진료 수수료, 건강계좌는 건강 등급에 따른 공공 보건소 PoC 수주와 기업 복지 구독입니다.

### Q. "카카오헬스케어와는?"
**A**: 카카오는 당뇨병이라는 이미 진단된 환자의 혈당 관리에 특화되어 있습니다. 건강계좌는 건강한 개인부터 만성질환 고위험군까지 전체 인구를 타겟으로, 검진 정상범위 이탈도 기반 규칙 점수를 제공합니다. 카카오는 의료 기기(CGM) 구매가 필수이지만, 건강계좌는 건강정보고속도로와 스마트폰 센서로 시작 가능합니다.

### Q. "룰루메딕, b.well 같은 글로벌 플레이어와 경쟁할 수 있나?"
**A**: 룰루메딕은 의료 마이데이터 기반 B2B 플랫폼이고, b.well은 미국 의료 기록 통합에 특화되어 있습니다. 건강계좌는 한국 공공 인프라(건강정보고속도로)를 기반으로 지역 보건소 PoC부터 시작하므로 공공 정책 연계성이 다릅니다. 글로벌 회사들이 한국의 소규모 보건소 사업까지 진입하기는 구조적으로 어렵습니다.
